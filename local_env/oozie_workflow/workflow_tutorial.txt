=== KỊCH BẢN WORKFLOW CHỐT CUỐI: TỰ ĐỘNG HÓA TỪ CÀO DỮ LIỆU ĐẾN KAFKA ===
============== (CẬP NHẬT KIẾN TRÚC DELTA LOAD - CHỐT THEO BIMI) =============

Anh em chú ý, luồng tự động Oozie đã được thiết kế lại toàn diện thành 5 BƯỚC siêu gọn nhẹ. Kiến trúc hiện tại hoạt động theo chuẩn Delta Load: Cào dữ liệu mới -> MySQL (Ghi đè) -> HDFS Raw (Ghi đè) -> HDFS Clean (Ghi nối).

Khi Quân Bảo bấm kích hoạt Oozie, hệ thống sẽ chạy liên hoàn 5 bước sau:

[Bước 1] Cào dữ liệu mới & Ghi đè trạm MySQL (Python Script - Quang Duy)
- Nhiệm vụ: Oozie gọi file scrape_and_insert.py.
- Cơ chế: Quét bảng tbl_bank_list xem Web yêu cầu cào mã nào -> CHỈ CÀO DỮ LIỆU NGÀY HÔM NAY (tránh nhân đôi lịch sử) -> Tạo file CSV tạm để chống tràn RAM -> Dùng lệnh TRUNCATE xóa sạch bảng MySQL cũ -> Đẩy dữ liệu mới vào bảng tbl_raw_stock -> Xóa file CSV tạm.
- Ý nghĩa: Bảng tbl_raw_stock lúc này chỉ là Trạm trung chuyển (Staging Area), KHÔNG chứa lịch sử mà chỉ chứa dữ liệu tươi mới nhất của ngày hôm đó để làm nhẹ máy ảo.

[Bước 2] Kéo dữ liệu lên HDFS Raw (Sqoop Import - Phúc An)
- Nhiệm vụ: Kéo dữ liệu từ trạm MySQL lên HDFS.
- Cơ chế: Oozie tự động xóa sạch thư mục Raw cũ trên HDFS, sau đó Sqoop Import dữ liệu của ngày hôm nay từ MySQL lên. (Cơ chế GHI ĐÈ)
- Đường dẫn đích chuẩn: /user/hadoop/stock_raw/

[Bước 3] Tiền xử lý và Tích lũy kho lịch sử (Apache Hive - Quang Duy)
- Nhiệm vụ: Oozie gọi script clean_data_advanced.hql của Duy.
- Cơ chế: Đọc dữ liệu Raw của ngày hôm nay, làm sạch, ép kiểu. Sau đó tiến hành GHI NỐI (Append) phần dữ liệu sạch này vào kho HDFS Cleaned. Kho HDFS Cleaned chính là "Két sắt lịch sử" của toàn hệ thống, ngày càng to ra.
- Đường dẫn đích chuẩn: 
  * /user/hadoop/stock_cleaned/ (Dạng Parquet cho Spark)
  * /user/hadoop/stock_cleaned_csv/ (Dạng CSV cho MapReduce)

[Bước 4] Tính toán đồng loạt 10 Task bằng Hadoop Streaming (MapReduce - Cả nhóm)
- Nhiệm vụ: Kích hoạt kịch bản run_all_mapreduce.sh.
- Cơ chế: Quét TOÀN BỘ kho dữ liệu sạch CSV. Hệ thống kích hoạt đồng thời 10 cụm Mapper/Reducer để tính 10 bài toán khác nhau. Kết quả xuất ra chuẩn JSON.
- Đường dẫn đích chuẩn: /user/hadoop/stock_result/result_mr_[task]/

[Bước 5] PySpark gom dữ liệu & Bắn qua Kafka (Spark + Kafka - Cả nhóm)
- Cải tiến cốt lõi: Thay vì 10 MapReduce tranh nhau bắn Kafka, PySpark sẽ đứng ra làm quản lý.
- Cơ chế: Oozie gọi file spark_join_kafka.py. Spark sẽ dùng tốc độ RAM quét sạch 10 thư mục kết quả ở Bước 4, gom lại thành một cục Dataframe duy nhất, sau đó biến thành Message đẩy lên Kafka Topic. Phía ngoài, Kafka Consumer nhặt tin và đổ vào bảng MySQL tbl_stock_analysis.

---------------------------------------------------------
YÊU CẦU BẮT BUỘC VÀ PHÂN CÔNG TỪNG THÀNH VIÊN KHI RÁP CODE
---------------------------------------------------------

1. QUÂN BẢO (Kiến trúc & Điều phối):
- Chuẩn bị sẵn máy ảo Local (Bật Hadoop, Oozie, MySQL, Kafka, Spark).
- Quản lý file Oozie workflow.xml. Lưu ý ở Node Hive ĐÃ XÓA thẻ <prepare> (thẻ delete folder), anh em KHÔNG được tự ý thêm vào kẻo Oozie xóa mất sạch dữ liệu lịch sử trên HDFS.

2. QUANG DUY (Thu thập & Tiền xử lý):
- File clean_data_advanced.hql: BẮT BUỘC phải dùng lệnh INSERT INTO TABLE (để Ghi nối/Append). TUYỆT ĐỐI KHÔNG dùng INSERT OVERWRITE, nếu không toàn bộ dữ liệu cào các ngày trước đó sẽ bị bay màu hoàn toàn.

3. PHÚC AN (Luân chuyển dữ liệu):
- Sqoop cứ chạy lệnh Import bình thường, KHÔNG CẦN tham số --append. Việc xóa và ghi đè Raw Data thì Oozie đã tự lo.
- Chính thức NGỪNG phát triển tác vụ Sqoop Backup ở cuối luồng.

4. CẢ NHÓM (Tính toán MapReduce):
- Tất cả các file Reducer nộp cho Bảo BẮT BUỘC phải xuất ra định dạng chuỗi JSON {"symbol": "...", "value": ...}. Cấm dùng định dạng Tab hay Phẩy.

5. BÌNH MINH & THƯỞNG (Giao tiếp & Ứng dụng - Chạy độc lập ngoài Oozie):
- Bình Minh: Viết file Kafka Consumer chạy ngầm hứng JSON từ Spark thả vào MySQL. Cấu hình Apache Drillbit làm cổng truy vấn xuyên HDFS và MySQL.
- Thưởng: Thiết kế Web Streamlit đảm nhận 2 vai trò: 
  1. Điều khiển hệ thống: Có form CRUD (Thêm/Sửa/Xóa) mã ngân hàng, ghi thẳng xuống bảng tbl_bank_list trong MySQL để hệ thống biết đường cào.
  2. Trực quan hóa: Kết nối qua Apache Drill/MySQL để bốc dữ liệu lên vẽ Dashboard.
