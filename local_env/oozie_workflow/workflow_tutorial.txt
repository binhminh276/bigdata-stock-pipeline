=== KỊCH BẢN WORKFLOW CHỐT CUỐI: TỰ ĐỘNG HÓA TỪ KHÂU CÀO DỮ LIỆU ĐẾN BACKUP ===
=================== (CẬP NHẬT: CƠ CHẾ TÍCH LŨY LỊCH SỬ VÀ KAFKA) ===================

Anh em chú ý, luồng tự động Oozie đã được thiết kế lại toàn diện, bọc luôn cả file Python cào dữ liệu vào vị trí đầu tiên và chuyển sang cơ chế lưu trữ tích lũy (giữ lịch sử, không xóa file cũ). Khi Quân Bảo bấm kích hoạt, hệ thống sẽ chạy liên hoàn qua 6 bước sau:

[Bước 1] Cào dữ liệu & Đổ vào trạm chứa tạm (Python Script - Quang Duy)
- Nhiệm vụ: Oozie gọi file "scrape_and_insert.py".
- Cơ chế: File này sẽ gọi API của vnstock/CafeF lấy dữ liệu mới nhất của 5 ngân hàng, sau đó mở kết nối trực tiếp vào MySQL Local, dùng lệnh "INSERT IGNORE" hoặc "ON DUPLICATE KEY UPDATE" để đẩy vào bảng tbl_raw_stock. 
- Ý nghĩa: Bảng tbl_raw_stock lúc này đóng vai trò là Trạm chứa tạm (Staging Area) trước khi đưa vào Hadoop.

[Bước 2] Kéo dữ liệu tích lũy lên Data Lake (Sqoop Import - Phúc An)
- Nhiệm vụ: Kéo dữ liệu thô từ MySQL lên HDFS.
- Cơ chế: Chạy lệnh Sqoop Import với tham số "--append". Lệnh này sẽ không xóa thư mục cũ mà tự động nối tiếp dữ liệu mới vào đuôi file cũ để giữ lại lịch sử vĩnh viễn.
- Đường dẫn đích: /user/quangduy/raw_data/

[Bước 3] Tiền xử lý và làm sạch dữ liệu (Apache Hive - Quang Duy)
- Nhiệm vụ: Oozie gọi file kịch bản "clean_data_advanced.hql" của Duy.
- Cơ chế: Đọc file thô từ thư mục của Duy, thực hiện lọc rác, xóa Null, ép kiểu dữ liệu. Kết quả được ghi dồn (INSERT INTO) để tích lũy lịch sử ngày này qua ngày khác.
- Đường dẫn đích: /user/quangduy/cleaned_data/

[Bước 4] Tính toán chỉ số tài chính Big Data (MapReduce Python - Cả nhóm)
- Nhiệm vụ: Hadoop Streaming kích hoạt bộ đôi "mapper.py" và "reducer.py" (file tổng hợp thuật toán của cả 5 người).
- Cơ chế: Đọc dữ liệu sạch tích lũy từ thư mục của Duy để tính toán: Giá đóng cửa trung bình, Tổng khối lượng, Độ biến động giá, và Đường trung bình động SMA. Do MapReduce yêu cầu thư mục đầu ra chưa tồn tại, bước này vẫn giữ thẻ <delete> để dọn rác thư mục kết quả tạm trước khi chạy.
- Đường dẫn đích: /user/hadoop/stock_result/

[Bước 5] Đẩy kết quả qua hàng đợi thông điệp (Apache Kafka Stream - Cả nhóm)
- Cải tiến cốt lõi: Thay vì dùng Sqoop Export đẩy thẳng vào MySQL dễ gây nghẽn/sập database, hệ thống chuyển sang dùng kiến trúc thông điệp trung gian Kafka.
- Cơ chế: Oozie gọi file "kafka_producer.py" để đọc kết quả từ thư mục stock_result, biến dữ liệu thành các Event Message rồi bắn vào Kafka Topic. Ở bên ngoài, file Kafka Consumer (chạy ngầm 24/7) sẽ nhặt các tin nhắn này ra và INSERT/UPDATE vào bảng kết quả "tbl_stock_analysis" trong MySQL.

[Bước 6] Sao lưu dự phòng an toàn (Sqoop Import Backup - Phúc An)
- Nhiệm vụ: Tạo bản sao dự phòng cho dữ liệu phân tích ngay trên Hadoop.
- Cơ chế: Chạy lệnh Sqoop Import với tham số "--append" để kéo ngược bảng kết quả tbl_stock_analysis từ MySQL về lưu trữ tích lũy lâu dài trên HDFS.
- Đường dẫn đích: /user/hadoop/stock_backup/


------------------------------------------------------------------------------------
YÊU CẦU BẤT BUỘC VÀ PHÂN CÔNG CHO TỪNG THÀNH VIÊN KHI RÁP CODE
------------------------------------------------------------------------------------

1. QUÂN BẢO (Kiến trúc & Điều phối):
- Chuẩn bị sẵn máy ảo Local (Bật Hadoop, Oozie, MySQL, Kafka).
- Tạo sẵn các thư mục trống trên HDFS (/user/quangduy/raw_data, /user/quangduy/cleaned_data, v.v.).
- Chép file "mysql-connector-java.jar" vào thư mục lib của Oozie trên HDFS.
- Gom toàn bộ file script của mọi người nộp lại và đưa lên thư mục: /user/hadoop/scripts/

2. QUANG DUY (Thu thập & Tiền xử lý):
- Viết file "scrape_and_insert.py": Phải import thư viện mysql.connector (hoặc pymysql) để kết nối và INSERT dữ liệu vào bảng tbl_raw_stock.
- Viết file "clean_data_advanced.hql": Đọc dữ liệu từ đường dẫn mới. TUYỆT ĐỐI KHÔNG dùng lệnh "INSERT OVERWRITE", bắt buộc phải dùng lệnh "INSERT INTO TABLE ..." để giữ lại lịch sử tích lũy trên HDFS.

3. PHÚC AN (Luân chuyển & Sao lưu):
- Cấu hình lệnh Sqoop Import ở Bước 2 và Bước 6: Phải cài thêm tham số "--append" ở cuối dòng lệnh.

4. BÌNH MINH & THƯỞNG (Lớp Giao tiếp & Ứng dụng độc lập ngoài Oozie):
- Đoạn code thuật toán MapReduce (Tính giá trung bình của Minh, tính độ biến động của Thưởng) bắt buộc phải nộp cho Bảo để gộp chung vào file chạy lõi.
- Ngoài luồng Oozie chạy cuối ngày: 
  + Bình Minh phụ trách bật dịch vụ Apache Drillbit 24/7, cấu hình kết nối HDFS và MySQL để làm cổng truy vấn SQL phân tán.
  + Thưởng phụ trách bật Web Streamlit 24/7, kết nối xuyên qua cổng Apache Drill của Minh để lấy kết quả từ bảng tbl_stock_analysis lên vẽ biểu đồ Candlestick và đường SMA.


DANH SÁCH FILE BẮT BUỘC PHẢI CÓ TRÊN THƯ MỤC /user/hadoop/scripts/
------------------------------------------------------------------------------------
1. workflow.xml (File cấu hình luồng tự động)
2. hive-site.xml (File cấu hình kết nối của Hive)
3. scrape_and_insert.py (Code cào dữ liệu của Duy)
4. clean_data_advanced.hql (Script Hive làm sạch của Duy)
5. mapper.py (Code Map chung của nhóm)
6. reducer.py (Code Reduce chung của nhóm)
7. kafka_producer.py (Code Python đẩy dữ liệu vào Kafka của nhóm)
