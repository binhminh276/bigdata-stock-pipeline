=== KỊCH BẢN WORKFLOW CHỐT CUỐI: TỰ ĐỘNG HÓA TỪ KHÂU CÀO DỮ LIỆU ĐẾN BACKUP ===

Anh em chú ý, luồng Oozie đã được thiết kế lại để bọc luôn cả file Python cào dữ liệu vào vị trí đầu tiên. Khi Quân Bảo bấm chạy Oozie, nó sẽ tự động kích hoạt tiến trình 6 bước liên hoàn không cần con người can thiệp:

[Bước 1] Cào dữ liệu & Insert MySQL (Quang Duy)
- Nhiệm vụ: Oozie gọi file python có tên là scrape_and_insert.py.
- Cơ chế: File này sẽ gọi API của vnstock/CafeF, lấy dữ liệu chứng khoán mới nhất về, sau đó MỞ KẾT NỐI trực tiếp đến MySQL Local và dùng lệnh "INSERT IGNORE" tống dữ liệu vừa cào được vào bảng tbl_raw_stock.

[Bước 2] Kéo dữ liệu gốc lên HDFS (Sqoop Import - Phúc An)
- Sau khi có dữ liệu trong MySQL, Oozie lập tức gọi Sqoop chui vào bảng tbl_raw_stock đó kéo ngược dữ liệu lên Hadoop.
- Đường dẫn đích: /user/quangduy/raw_data

[Bước 3] Tiền xử lý dữ liệu (Apache Hive - Quang Duy)
- Oozie gọi file clean_data_advanced.hql để dọn dẹp các giá trị rác, dữ liệu Null. 
- Đường dẫn đích: /user/quangduy/cleaned_data

[Bước 4] Tính toán chỉ số tài chính (MapReduce Python - Cả nhóm)
- Oozie gọi bộ đôi mapper.py và reducer.py để tính Trung bình giá, Khối lượng tổng, Độ biến động và đường trung bình động SMA. 
- Đường dẫn đích: Kết quả xuất ra /user/hadoop/stock_result

[Bước 5] Đẩy kết quả qua Kafka Stream (Nâng cấp thay Sqoop Export)
- Oozie gọi file kafka_producer.py. File này nhặt kết quả từ MapReduce, băm thành message rồi ném vào hệ thống hàng đợi Kafka. Đầu bên kia sẽ có Kafka Consumer chạy ngầm tự động lấy và Insert/Update vào bảng tbl_stock_analysis trong MySQL.

[Bước 6] Sao lưu dự phòng an toàn (Sqoop Import - Phúc An)
- Để tránh mất trắng dữ liệu nếu database sập, Sqoop sẽ kéo ngược bảng kết quả đó từ MySQL về cất trong thư mục an toàn.
- Đường dẫn đích: /user/hadoop/stock_backup
