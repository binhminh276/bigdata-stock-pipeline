=== KỊCH BẢN WORKFLOW CHỐT CUỐI: TỰ ĐỘNG HÓA TỪ CÀO DỮ LIỆU ĐẾN KAFKA ===
================ (CẬP NHẬT MỚI: BỎ BACKUP SQOOP - DÙNG SPARK GOM DATA KAFKA) ================

Anh em chú ý, luồng tự động Oozie đã được thiết kế lại toàn diện thành 5 BƯỚC siêu gọn nhẹ. Điểm nâng cấp lớn nhất là loại bỏ thao tác Backup MySQL vô nghĩa (vì HDFS đã là pháo đài an toàn), đồng thời đưa PySpark vào làm chốt chặn cuối để gom 10 luồng MapReduce tránh nghẽn cổ chai.

Khi Quân Bảo bấm kích hoạt, hệ thống sẽ chạy liên hoàn 5 bước sau:

[Bước 1] Cào dữ liệu & Đổ vào trạm chứa tạm (Python Script - Quang Duy)

Nhiệm vụ: Oozie gọi file scrape_and_insert.py.

Cơ chế: Cào API của vnstock/CafeF lấy dữ liệu mới nhất, sau đó mở kết nối trực tiếp vào MySQL Local, dùng lệnh INSERT IGNORE đẩy vào bảng tbl_raw_stock.

Ý nghĩa: Bảng tbl_raw_stock đóng vai trò là Trạm chứa tạm (Staging Area). Cuối tuần anh em tự chạy lệnh TRUNCATE dọn dẹp MySQL cho nhẹ máy.

[Bước 2] Kéo Raw Data lên HDFS theo cơ chế XÓA CŨ - CHỨA MỚI (Sqoop Import - Phúc An)

Nhiệm vụ: Kéo dữ liệu thô từ trạm MySQL lên HDFS.

Cơ chế: Để tránh tràn ổ cứng, Oozie sẽ tự động xóa sạch thư mục Raw cũ trên HDFS trước khi Import dữ liệu mới.

Đường dẫn đích chuẩn: /user/hadoop/stock_raw/

[Bước 3] Tiền xử lý và lưu TÍCH LŨY LỊCH SỬ (Apache Hive - Quang Duy)

Nhiệm vụ: Oozie gọi script clean_data_advanced.hql của Duy.

Cơ chế: Đọc file thô vừa kéo về, lọc rác, ép kiểu. Kết quả được ghi dồn (INSERT INTO TABLE) nối đuôi vào kho lịch sử. Đây chính là "trái tim" Data Lake. Dữ liệu sạch xuất ra 2 định dạng: Parquet (cho Spark) và CSV (cho MapReduce).

Đường dẫn đích chuẩn: * /user/hadoop/stock_cleaned/ (Dạng Parquet)

/user/hadoop/stock_cleaned_csv/ (Dạng CSV)

[Bước 4] Tính toán đồng loạt 10 Task bằng Hadoop Streaming (MapReduce - Cả nhóm)

Nhiệm vụ: Kích hoạt kịch bản run_all_mapreduce.sh.

Cơ chế: Quét TOÀN BỘ kho dữ liệu sạch CSV. Hệ thống kích hoạt đồng thời 10 cụm Mapper/Reducer để tính 10 bài toán khác nhau. Kết quả xuất ra chuẩn JSON.

Đường dẫn đích chuẩn: /user/hadoop/stock_result/result_mr_[task]/

[Bước 5] PySpark gom dữ liệu & Bắn qua Kafka (Spark + Kafka - Cả nhóm)

Cải tiến cốt lõi: Thay vì 10 MapReduce tranh nhau bắn Kafka, PySpark sẽ đứng ra làm quản lý.

Cơ chế: Oozie gọi file spark_join_kafka.py. Spark sẽ dùng tốc độ RAM quét sạch 10 thư mục kết quả ở Bước 4, gom lại thành một cục Dataframe duy nhất, sau đó biến thành Message đẩy lên Kafka Topic. Phía ngoài, Kafka Consumer nhặt tin và đổ vào bảng MySQL tbl_stock_analysis.

YÊU CẦU BẮT BUỘC VÀ PHÂN CÔNG TỪNG THÀNH VIÊN KHI RÁP CODE
1. QUÂN BẢO (Kiến trúc & Điều phối):

Chuẩn bị sẵn máy ảo Local (Bật Hadoop, Oozie, MySQL, Kafka, Spark).

Đảm bảo cấu trúc thư mục máy ảo chuẩn xác: Code ngoài Home (/home/hadoop/), code MR vào thư mục mapreduce/, code Spark vào bigdata_spark/.

Quản lý file Oozie workflow.xml bản 89 dòng mới nhất.

2. QUANG DUY (Thu thập & Tiền xử lý):

File clean_data_advanced.hql: TUYỆT ĐỐI KHÔNG dùng INSERT OVERWRITE, bắt buộc dùng INSERT INTO TABLE để đổ dồn dữ liệu sạch. Phải đảm bảo tạo ra cả bảng Parquet và bảng CSV phân tách bằng dấu phẩy.

3. PHÚC AN (Luân chuyển dữ liệu):

Lệnh Sqoop ở Bước 2: BỎ tham số --append (hệ thống sẽ tự động ghi đè Raw Data theo thẻ delete của Oozie).

Chính thức NGỪNG phát triển tác vụ Sqoop Backup ở cuối luồng.

4. CẢ NHÓM (Tính toán MapReduce):

Tất cả các file Reducer nộp cho Bảo BẮT BUỘC phải xuất ra định dạng chuỗi JSON {"symbol": "...", "value": ...}. Đầu ra cấm dùng định dạng Tab hay Phẩy.

5. BÌNH MINH & THƯỞNG (Giao tiếp & Ứng dụng - Chạy độc lập ngoài Oozie):

Bình Minh: Viết file Kafka Consumer chạy ngầm hứng JSON từ Spark thả vào MySQL. Cấu hình Apache Drillbit làm cổng truy vấn xuyên HDFS và MySQL.

Thưởng: Thiết kế Web Streamlit kết nối qua Drill/MySQL để lấy dữ liệu vẽ Dashboard.

DANH SÁCH 6 FILE/THƯ MỤC CỐT LÕI BẮT BUỘC PHẢI CÓ
Anh em lưu ý, toàn bộ code phải tuân thủ đúng tên và vị trí này trên máy ảo:

/home/hadoop/bigdata_spark/spark_join_kafka.py: Code PySpark gom 10 file kết quả và làm Producer đẩy lên Kafka.
